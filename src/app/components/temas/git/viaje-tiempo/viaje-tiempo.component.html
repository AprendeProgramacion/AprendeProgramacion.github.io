Ya sabemos guardar una version de nuestro proyecto. Pero, ¿Para qué?

Con Git podemos viajar hacia una version anterior en caso de necesitarlo, por ejemplo, modifiqué algo y otra funcionalidad se dañó. puedo ir a ua version anterior en la que si funcionaba y saber que fue lo que se dañó.

Antes del comando para viajar en el tiempo, necesitamos obtener el hash del commit al que vamos a ir, para eso usamos:

  • git log
    •

    Tambien necesitamos el nombre de la rama en la que nos encontramos actualmente. aparece al lado de un commit de la siguiente manera:

    Si todo sale bien, debe salir un mensaje y al final de dicho mensaje debe decir algo como:

  • (HEAD -> {{'<'}}rama{{'>'}})

    • Una vez tengamos el hash alfanumerico, podemos ir a ese commit usando:

  • git checkout {{'<'}}hash{{'>'}}

    • Si todo sale bien, debe salir un mensaje y al final de dicho mensaje debe decir algo como:

  • Now HEAD is on ({{'<'}}hash{{'>'}})

    • para volver al presente hay que tener en cuenta el nombre de la rama en la que estabamos antes:

  • git checkout {{'<'}}rama{{'>'}}
