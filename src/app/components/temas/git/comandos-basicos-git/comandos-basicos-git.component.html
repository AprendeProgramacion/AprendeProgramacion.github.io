Una vez iniciado el repositorio, podemos empezar a guardar cambios. Esto podríamos hacerlo mediante interfaz grafica, pero como git fue creado para trabajar con linea de comandos, si usamos una GUI no tendremos todas las funcionalidades que nos ofrece.


Ver estado del proyecto

Ya hicimos los cambios en nuestro proyecto, ahora hay que guardarlos. Pero antes de eso tenemos que mirar que archivo o archivos han sido agregados, modificados o eliminados. Para eso debemos usar el comando:

  • git status
    •

    Este comando muestra el estado de los archivos, que pueden ser 'Untracked', 'Modified' y 'Staged'.


    Agregar cambios a staging

    Si queremos guardar un archivo que haya sido modificado o recien creado, primero tenemos que agregarlo al area de staging, para ello usamos:

  • git add {{'<'}}archivo 1{{'>'}} {{'<'}}archivo 2{{'>'}} ...
    •

    Este comando nos permite agregar los archivos que le especifiquemos, sin embargo podemos ahorrarnos el escribir muchos archivos si solo queremos agregarlos todos. Para esto usamos:

  • git add .
    •

    Recordemos que el punto nos indica que seleccionamos todo en el directorio en el que nos encontramos.


    Guardar cambios como una version

    cuando ejecutamos 'git add', estamos agregando archivos a un lugar temporal, no los estamos guardando en la base de datos.

    Cuando todos los archivos que queremos guardar se encuentran en staging, podemos usar:

  • git commit -m {{'"<'}}mensaje{{'>"'}}
    •

    Este comando guarda todo lo que teniamos en staging como una nueva version, para que podamos acceder a ella en caso de necesitarlo.

    El argumento '-m' indica que a continuacion vamos a introducir un mensaje, que debe ir entre comillas dobles ({{'"'}}). Si este no se indica, git abrirá vim y solicitará el mensaje.


    Ver lista de versiones

    Podemos ver una lista con todos los commits que se han hecho a modo de historial con el comando:

  • git log
