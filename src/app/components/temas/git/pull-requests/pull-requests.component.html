Pull Requests

Cuando trabajamos con GitHub, podemos simplemente mezclar ramas en git usando el comando 'git merge'. Pero esto no es buena practica ya que muchas veces deberiamos poder hacer revision de codigo para que uno o varios desarroladores nos den sugerencias sobre algo que est√© mal hecho o simplemente para agregar o modificar algo.

Para esto, Github cuenta con una herramienta llamada Pull Request. En lugar de hacer un merge directo, lo que hace es crear una especie de tarea en la que pide a otros desarrolladores que revisen el codigo, cuenta tambien con un chat en el que se pueden hacer sugerencias, solicitar cambios, entre otros.
