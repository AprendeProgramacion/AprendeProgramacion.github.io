Pensemos el historial de git como una linea de tiempo, cambio tras cambio podemos notarlo. Pero git nos da la posibilidad de separarlo.

Entramos a un concepto nuevo, las 'ramas' nos permiten separarnos de la linea de tiempo principal para trabajar de manera aislada. Esto nos permite hacer cambios sin afectar el historial principal (normalmente conocido como area de produccion) para que, si nos equivocamos, podemos estar seguros que el area de producción está a salvo.


Crear Ramas

Para crear nuestra rama ejecutamos:

  • git branch {{'<'}}rama{{'>'}}
    •

    Podemos ver las ramas que tenemos con:

  • git branch
    •

    Aparecen todas las ramas locales. Podemos identificar la rama en la que nos encontramos porque tiene un asterisco (*).

    Para movernos de una rama a otra podemos ejecutar:

  • git checkout {{'<'}}rama a movernos{{'>'}}
    •

    Podemos cambiar el nombre de la rama en la que estamos, usando:

  • git branch -M {{'<'}}nuevo nombre{{'>'}}
    •

    Para ver los logs mas organizados cuando trabajamos con ramas podemos usar una version modificada de 'git log':

  • git log --oneline --graph --decorate
    •

    git reset

    Este comando nos permite volver a un commit anterior, pero se elimina todo el historial de los cambios posteriores a el.


    Fusion de ramas

    Si queremos fusionar dos ramas, debemos movernos a la rama que va a recibir los cambios, luego ejecutar:

  • git merge {{'<'}}rama{{'>'}}
