¿Qué es SSH?

SSH (Secure SHell) es un protocolo de conexion para enviar y recibir datos de manera cifrada a traves de internet. funciona usando llaves.

Funciona de la siguiente manera: Nuestro pc le envia al receptor de mis archivos (en este caso GitHub) una llave y él hace lo mismo. Cada llave tiene su pareja, que cada uno conserva.

Al momento de enviar informacion, mi pc debe cifrarla usando la llave que le dieron, cuando el receptor obtiene el mensaje cifrado, lo puede abrir utilizando la pareja de la llave que envió. Sucede de igual manera cuando el pc recibe informacion.


Crear llaves ssh y configurar conexion

Vamos a crear una conexion SSH para enviar y recibir informacion de GitHub al publicar el proyecto.

Lo primero que debemos hacer es movernos a la carpeta 'HOME' (En la terminal se muestra '~'). Para eso ejecutamos el comando:

  • cd
    •

    Sin argumentos, esto nos llevará a '~'

    Una vez en '~', ejecutamos:

  • ssh-keygen -t rsa -b 4096 -C {{'<'}}correo de github{{'>'}}
    •

    Nos va a pedir una ruta, por conveniencia la dejamos vacia. Luego nos va a pedir crear una contraseña, ya depende de la seguridad que deseen, en mi caso la dejo vacia

    Si las llaves se crearon correctamente, debería aparecer una imagen de caracteres, esa image n es un fingerprint de las llaves.

    Ahora debemos verificar si el servidor SSH está funcionando correctamente, para esto ejecutamos:

  • eval $(ssh-agent -s)   # Windows & Linux
  • eval {{'"'}}$(ssh-agent -s){{'"'}} # MAC
    •

    Si todo sale bien, debe aparecer 'agent pid 123' donde el numero puede ser cualquiera.

    Despues de eso debemos movernos a la carpeta '.ssh' que se creó en 'HOME':

  • cd ~/.ssh/
    •

    Una ves ahi, ejecutamos 'ls' y nos debe salir algo como esto:

  • id_rsa
  • id_rsa.pub
    •

    Debemos agregar nuestra llave privada (id_rsa) al computador, para eso ejecutamos:

  • ssh-add id_rsa
    •

    Si todo sale bien, debe aparecer 'Identity added ({{'<'}}correo{{'>'}})
