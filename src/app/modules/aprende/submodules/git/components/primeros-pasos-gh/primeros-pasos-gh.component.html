Primeros pasos en GitHub

Cuando hacemos un cambio en el repositorio con Git, lo estamos guardando en local, es decir, nuestro computador. Pero Â¿que sucede si despues queremos publicar esos cambios?

Para eso existe el comando 'push' que nos permite subir el historial de una rama al repositorio remoto (GitHub).

Para usar el comando, primero debemos tener el repositorio local vinculado al repositorio en GitHub.

  • git remote add {{'<'}}nuevo nombre{{'>'}} {{'<'}}URL{{'>'}}
    •

    Una vez vinculado el repositorio local y remoto, cuando hagamos un nuevo commit, si queremos enviar los cambios, basta con ejecutar:

  • git push {{'<'}}nombre remoto{{'>'}} {{'<'}}nombre rama{{'>'}}
    •

    El nombre de remoto es el nombre que le pusimos al repositorio cuando agregamos la url para vincularlo con GitHub. Por lo general es 'origin'.

    El nombre de la rama debe ser exactamente igual a la rama en la que te encuentras, puedes ver el nombre usando:

  • git branch
